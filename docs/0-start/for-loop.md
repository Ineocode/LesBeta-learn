# for 循环

::: info 一句话定义
当你需要把同一件事重复做「已知次数」时，for 循环让你用三行代码代替成百上千行重复。
:::

---

## ① 为什么你需要这个

假设老师让你输出从 1 到 100 的所有数字。

你当然可以写 100 行 `cout`：

```cpp
cout << 1 << "\n";
cout << 2 << "\n";
// ……再写 98 行
```

但如果老师改成「输出 1 到 1000000」呢？没有人会去写一百万行代码。**只要你发现自己在复制粘贴几乎一样的代码，就说明你需要循环了。** for 循环就是为「重复次数已知」这种场景量身定做的工具。

---

## ② 核心概念

for 循环把三件事写在同一行括号里，用分号隔开：

```cpp
for (初始化; 继续条件; 每轮更新) {
    循环体；   // 这里的代码会被重复执行
}
```

执行顺序是这样的：先做一次**初始化**；然后检查**继续条件**，成立就执行一次循环体；每执行完一轮，做一次**更新**，再回去检查条件。条件不成立时，整个循环结束。

最小例子——输出 1 到 5：

```cpp
for (int i = 1; i <= 5; i++) {
    cout << i << "\n";
}
```

这里 `i` 从 1 开始，每轮 `i++` 加一，直到 `i <= 5` 不再成立（也就是 `i` 变成 6）时停下。输出是 1、2、3、4、5。

::: danger 反例：这样写会变成死循环
```cpp
for (int i = 1; i <= 5; i--) {   // 注意是 i--
    cout << i << "\n";
}
```
这里更新写成了 `i--`，`i` 越来越小，永远不会大于 5，条件永远成立。程序不会报错，但会**一直运行下去直到超时（TLE）**。这是新手最常见的坑之一——死循环不报错，只会卡住。
:::

::: tip 如果你学过 Python
Python 的 `for i in range(1, 6)` 输出的是 1 到 **5**（不含 6），因为 range 的上界是开区间。
C++ 对应的写法是 `for (int i = 1; i <= 5; i++)`，用 `i <= 5` 才能取到 5。
**上界差 1，是从 Python 迁移过来最容易错的地方。**
:::

## ③ 代码模板

**标准计数循环（竞赛中最常用，0-indexed）：**

```cpp
for (int i = 0; i < n; i++) {
    // 处理第 i 个元素，i 从 0 到 n-1
}
```

竞赛中习惯让下标从 0 开始、用 `i < n` 作为条件，这样和数组、STL 的下标完全一致，减少思维切换。

**逆序遍历：**

```cpp
for (int i = n - 1; i >= 0; i--) {
    // 从最后一个元素往前处理
}
```

**读入一个数组：**

```cpp
for (int i = 0; i < n; i++) {
    cin >> a[i];
}
```

**嵌套循环（遍历二维表格）：**

```cpp
for (int i = 0; i < n; i++)
    for (int j = 0; j < m; j++)
        cout << a[i][j] << " ";
```

## ⑤ 概念练习（L1）

### 第 1 题

下面代码输出什么？

```cpp
for (int i = 0; i < 3; i++) {
    cout << i << " ";
}
```

A. `1 2 3`　B. `0 1 2`　C. `0 1 2 3`　D. `1 2`

::: details 查看答案
**正确答案：B**

`i` 从 0 开始，条件是 `i < 3`，所以取到的值是 0、1、2，到 `i = 3` 时条件不成立就停了。这道题考的是「0-indexed + 小于号」的组合，是竞赛最常见的写法。
:::

### 第 2 题

下面这段代码会发生什么？

```cpp
for (int i = 1; i > 0; i++) {
    cout << i << "\n";
}
```

A. 输出 1 然后停止　B. 什么都不输出　C. 死循环（一直运行）　D. 编译错误

::: details 查看答案
**正确答案：C**

`i` 从 1 开始不断加一，`i > 0` 永远成立，循环停不下来。这就是死循环——程序不报错，但会一直运行到超时。
:::

### 第 3 题

要输出 5 到 1（倒着数），下面哪个对？

A. `for (int i = 5; i >= 1; i++)`
B. `for (int i = 5; i >= 1; i--)`
C. `for (int i = 1; i <= 5; i--)`
D. `for (int i = 5; i <= 1; i--)`

::: details 查看答案
**正确答案：B**

倒着数要从大到小，所以初始值是 5，每轮 `i--` 减一，条件是 `i >= 1`。A 用了 `i++` 会变死循环；D 的条件 `i <= 1` 一开始就不成立，一次都不执行。
:::

## ⑥ 编程练习（L2 + L3）

### L2 · 本地练习（在 IDE 中完成）

**练习：求 1 到 n 的和**

读入一个整数 n，用 for 循环计算 1 + 2 + 3 + … + n，输出结果。

测试：输入 100，应输出 5050。

::: details 参考代码
```cpp
#include <iostream>
using namespace std;
int main() {
    int n;
    cin >> n;
    long long sum = 0;   // 用 long long 防止 n 很大时溢出
    for (int i = 1; i <= n; i++) {
        sum += i;
    }
    cout << sum << "\n";
    return 0;
}
```
注意这里 `sum` 用了 `long long`——当 n 达到几万时，结果会超过 `int` 的范围。这是一个提前植入的「溢出」意识。
:::
